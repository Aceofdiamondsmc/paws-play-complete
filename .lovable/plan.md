`src/integrations/supabase/types.ts` still contains type definitions for `profiles_safe` and `profiles_safe_backup_test`. This file is auto-generated by the Supabase CLI from the live schema and **cannot and should not be manually edited**. It will be automatically regenerated the next time a database migration runs, at which point it will reflect the current schema (no deleted views, correct `public_posts` and `public_profiles` definitions already present).

No application code consumes these stale types at runtime â€” they are just dead TypeScript type declarations.
